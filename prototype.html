How NetVault-Tactical Works

A look at the system architecture, components, and hardware/software specifications behind the prototype.

System Architecture

NetVault-Tactical is composed of five core service components, each containerized and independently deployable:

Scheduler Engine

Manages backup job queues, schedules, and event-driven triggers across all registered nodes.

Collection Agent

Connects to network devices via SSH and SNMP to extract and normalize configuration data.

Encryption Worker

Applies AES-256-GCM encryption to every backup archive and manages TPM-backed key storage.

Sync Daemon

Monitors connectivity and pushes encrypted archives to the cloud (e.g., JWCC) when bandwidth permits.

Management Interface

Web-based dashboard for node registration, job monitoring, restoration, and audit log review.

Data Flow

[01] Tactical network devices (routers, switches, SATCOM terminals)
[02] Collection Agent pulls configuration via SSH / SNMP
[03] Encryption Worker secures the archive (AES-256-GCM)
[04] Stored on local edge server — available offline, immediately
[05] Sync Daemon pushes to cloud (JWCC) when bandwidth allows
[06] Operator restores via Management Interface in one click
